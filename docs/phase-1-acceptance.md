# 阶段 1 验收说明

## 范围

阶段 1 聚焦项目骨架、配置与系统入口，不包含 GitHub Actions、线上仓库浏览、团队评审流转等后续阶段能力。

## 已完成项

1. 可启动桌面应用：Tauri + Vite + React 项目骨架已建立，`npm run start` 可进入桌面端，`npm run dev` 可启动前端调试服务。
2. 主界面框架：左侧仓库入口、顶部命令栏、工作台内容区、右侧变更/评审区域、底部状态栏已经形成稳定布局。
3. 分层结构：前端已按 `components`、`hooks`、`lib` 分层；Rust 侧提供 Tauri command 作为系统能力入口，命令调用、路径处理、输出解码和错误返回统一收敛在后端服务层。
4. 命令执行基础能力：后端统一封装 `run_command` / `run_command_args`，补充 Windows 下 `npm.cmd` 与 SVN CLI 解析；耗时质量检查已通过后台任务执行，避免阻塞桌面端交互。
5. 本地配置读写：`useSettings` 使用本地持久化保存自动刷新、刷新间隔、默认变更视图和界面语言。
6. 模块接口草案：仓库、状态、文件浏览、diff、提交、ignore、分支和质量检查的数据结构集中定义在 `src/lib/api.ts`，为后续模块继续扩展保留稳定接口。
7. 国际化基础能力：`src/lib/i18n.ts` 建立中英文资源结构，设置页可切换 `zh-CN` / `en-US`，并同步更新文档 `lang` 属性；导航、顶部命令栏和设置页已接入翻译函数。
8. Win11 风格基础 UI：全局颜色、字号、间距、按钮、输入、状态徽标、弹窗、浅/深/系统主题和基础响应式布局已落地。

## 验证命令

```powershell
npm run build
npm run test:ui
cargo fmt --check
cargo check
```

## 已知边界

1. 国际化目前覆盖基础框架和设置入口，业务面板长文案会在后续阶段逐步迁移到同一资源结构。
2. Rust 后端当前仍在单个 `lib.rs` 中承载多类 command，阶段 1 已具备统一能力入口；后续功能膨胀时可按 VCS、系统集成和应用服务拆分模块。
3. 系统级 Windows 右键菜单、通知和命令行入口属于后续阶段，不纳入阶段 1 完成范围。
